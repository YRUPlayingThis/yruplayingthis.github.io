How I Started my First Github Website
(With No Experince)

As of 5-14-2025, I have just figured out how to launch and edit my own website and repository through the use of Git, and Github. It is now my intention to showcase to you, reader, how I managed to do it with no previous experience with Github, or web development!


  • Resources Used
  • Essential Github Setup
  • Optional Way to Edit Website Locally

Resources Used

  • (Required) Github
  • (Optional) Microsoft Studio Code
  • (optional) Git for Windows

Setting up with Github

In this section we will go over the bare minimum to host a github website, requiring only a device and the website itself

  1. Create your Github Account
  2. Create your first (or new) repository, name it:
    (your Username here).github.io
    Ensure that your github username is before the first period and in ALL lowercase characters!
  3. Go to the Main Branch and click Add file -> Create new file and call it index.html
  4. Go to your Repo Settings > Pages and then set your branch to main (in the root folder)
  5. If all well, your html code, and therefore webpage, is now hosted at
    https://(your username here).github.io

Small Optional steps I Followed

I personally took time to add a few different folders within my main directory with their own index.htmls within them to create multiple pages within my website.
If you wish to do the same, then when creating a new file, write
(folder name)/index.html
This creates a new default directory and the folder that contains it, visiting it with something such as (username)/github.io/(folder name) now takes the user to that page!

I also created a style.css file in the main directory, putting most of all my styling preferences on it and referring to it within all my pages' html code. Its as simple as adding a file and naming it style.css
This allows me to add the formating, colors, and text effects you see within this website now!

Editing your Github Repo Locally

These are now the steps that I myself followed to begin working on this very website from my pc. After these steps, you will get to code offline, check how your edits look before a commitment, and see your files in a neatly organized fashion. With these steps, life will get a lot easier to actively work on your repo!

  1. Download both Visual Studio Code, and Git for Windows from their websites
  2. Once Git has downloaded, set your Git identity by doing the following:
    git config --global user.name "name"
    git config --global user.email "your@email.com"
    Replace the quoted text with your name, and then email respectively
  3. Open Visual Studio Code and sign in with your GitHub Account
  4. Clone your repo into VS Code with the GitHub URL button by going to your repo, clicking on
    Code -> HTTPS and then copy the link right below the HTTPS
    In VS Code, open the command palette with Ctrl + Shift + P and type Git: Clone
    After pressing enter you should be able to paste your link in.
    You may also try to clone your repo link through the VS Code Source Control Tab on the left side of the program
  5. To test your files and site before committing your changes, click on Extensions on the left tab (bottom option) and search for Live Server Extension. This will allow you to right click on your html file and run it on a live server, seeing the web page to ensure your edits are correct before you committ your changes!
  6. Now you can work freely with your repos synced, simply make sure to save your files when done editing them
    (Ctrl + S) and when you're happy with your work, go to the Source Control tab and click commit (the message is a summary of your update). This updates your local repository.
    Press the Sync button in the same spot to then share your changes to your Github repo online and boom!
    You can now freely edit and review your repo on your local pc and then share changes to the website without even touching a web browser!
